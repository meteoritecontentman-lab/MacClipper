How to add new things
- Add a new website page: create a component in website/src/components and register it in website/src/App.js.
- Add a new website route: update website/src/App.js and make sure the page is reachable from the relevant nav or card.
- Add a new billing or entitlement action: start in website/src/lib/cloudAccount.js, then wire the UI in Support.js or Settings.js.
- Add a new app setting: put the state in Sources/MacClipper/AppModel.swift and render it in SettingsView.swift and MenuSettingsPage.swift.
- Add a new capture or export behavior: change Sources/MacClipper/ReplayBufferRecorder.swift first, then adjust AppModel.swift if state or policy changes are needed.
- Add a new bot command: update /Users/meteorite/MacClipperBot/bot.py and keep Pro changes tied to real entitlement updates.
- Add a new Firebase API route: update functions/src/routes and validate with npm run build.
- Add a new backend route: update backend/server.js or the related service file under backend/runtime and document the request/response shape.
- Add a new public-facing asset or icon: update scripts/generate_app_icon.swift or the relevant packaging script, then rebuild the app.

Echo / audio guidance
- Common echo cause: enabling System Audio while also selecting a loopback/monitor virtual microphone can double-capture audio.
- The current safeguard checks microphone names for terms like BlackHole, Loopback, Virtual, Monitor, Stereo Mix, System Mix, Aggregate, VB-Audio, Soundflower.
- If users still hear echo, the first things to test are:
  - Disable System Audio when using a loopback mic.
  - Use a physical microphone instead of a loopback/monitor input.
  - Confirm the selected mic is not a virtual monitor device.

  How to debug audio complaints
  - Check whether the user is recording System Audio, microphone, or both.
  - Check whether their selected mic is a loopback device.
  - Check whether the app is suppressing microphone capture because the device looks like a monitor/virtual input.
  - If the problem is still unclear, inspect ReplayBufferRecorder.swift and AppModel.swift together.
